# Structured Questions Skill — 让 AI 帮你问出你自己问不出来的问题

**适用工具**: Claude Code / Codex CLI
**安装指南**: [structured-questions-install.md](./structured-questions-install.md)

---

## 这个 Skill 解决什么问题？

你有没有遇过这种情况——

你叫 AI 帮你做一件事，它回了一大段文字，中间埋了三、四个需要你决定的事情，但你读完整段之后，反而更不确定该怎么回复。或者更常见的：**你根本不知道这件事有什么好问的，AI 就直接帮你决定了。**

这个 skill 做两件事：

1. **让 AI 主动帮你梳理思路。** 当你自己都还没想清楚要什么的时候，AI 会把混乱的局面拆成几个具体的选择题，每题附上选项、优缺点、和它的推荐——你不需要自己想出「该问什么」，AI 帮你问。

2. **把散落在长文里的决策点，收拢成一张清单。** 不再是「请看完以上分析后告诉我你的想法」这种丢球式回复。每个需要你拍板的事情独立列出，选项带优缺点，推荐项标出来——你扫一眼就能做决定。

---

## Before / After 对比

### 价值一：你还没想清楚的时候，AI 帮你拆解问题

> 你不需要先想好「该问什么」——AI 主动把模糊的想法拆成具体的选择题。

---

#### 场景 A：想办一场活动，但脑中只有「想办」两个字

**Before — AI 直接帮你决定一切：**

```
你：帮我规划一场活动

AI：好的！我建议办一场 30 人的周末工作坊，地点选在共享空间，
主题是「AI 工具入门」。我先帮你列出时间表...
```

问题：你只是「想办活动」，AI 就决定了人数、形式、主题、场地。你连「原来活动可以有这些类型」都没想过。

**After — AI 帮你从混乱中理出头绪：**

```
Q1: 这场活动的核心目标是什么？

  ✨ A) 教学 / 知识分享 (Recommended)
     让参加者学会一个具体技能
     😃：内容好准备，效果可衡量
     😫：需要你自己有足够专业度

  B) 社群经营 / 认识人
     让同领域的人互相连结
     😃：轻松、参加者压力低
     😫：效果难量化，容易流于闲聊

  C) 品牌曝光 / 行销
     让更多人知道你或你的产品
     😃：直接带来潜在客户
     😫：需要行销预算和推广渠道

Q2: 预计多少人参加？
  ✨ A) 10-15 人 (Recommended) ...
  B) 30-50 人 ...
  C) 100+ 人 ...
```

差异：你本来只有「想办活动」。AI 帮你厘清「目标」「人数」「形式」三件事——回答完，你就知道该办什么了。

#### 场景 B：想开始经营自媒体，但不知道从哪里开始

**Before — AI 替你选了平台和方向：**

```
你：我想开始经营自媒体

AI：建议你从 Instagram 开始，每周发 3 篇 Reels，主题聚焦在你的
专业领域。我帮你规划第一周的内容排程...
```

问题：你还在想「要不要做」，AI 已经在排第一周的发文了。

**After — AI 先帮你想清楚几个关键问题：**

```
Q1: 经营自媒体的主要动机是什么？

  ✨ A) 建立个人品牌 (Recommended)
     让业界知道你是谁、你擅长什么
     😃：长期累积，对职涯有复利效果
     😫：需要持续产出，短期看不到回报

  B) 变现 / 赚钱
     透过业配、课程、付费内容获利
     😃：有明确的收入目标可以追踪
     😫：初期没流量时收入几乎为零

  C) 纪录与分享
     把学到的东西写下来，顺便帮助别人
     😃：压力最低，做得开心就好
     😫：成长速度慢，容易半途而废

Q2: 你比较擅长哪种内容形式？
  ✨ A) 写文字 (Recommended) ...
  B) 拍影片 ...
  C) 录 Podcast ...
```

差异：AI 没有直接叫你「去开 IG 帐号」，而是先帮你想清楚「为什么做」和「用什么形式」。

#### 场景 C：想做一个 Side Project，但还没想清楚方向

**Before — AI 直接帮你决定做什么、用什么技术：**

```
你：帮我做一个 side project

AI：好的！我建议做一个个人博客网站。我会用 Next.js + MDX 来建，
搭配 Tailwind CSS 做样式，部署到 Vercel。先初始化项目...
```

问题：你只是想做「一个 side project」，AI 决定了产品、框架、部署。你连「原来还有这些选择」都不知道。

**After — AI 帮你把模糊的念头拆成三个选择题：**

```
Q1: 这个 Side Project 的目标是什么？

  ✨ A) 学新技术 (Recommended)
     做完学会一个新框架或语言
     😃：成就感高，履历有东西写
     😫：可能做不完

  B) 解决自己的痛点
     做一个自己每天会用的小工具
     😃：有动力做完，因为自己要用
     😫：范围容易膨胀

  C) 建立作品集
     做一个能展示给面试官看的东西
     😃：求职直接加分
     😫：需要花时间打磨 UI

Q2: 你愿意投入多少时间？
  ✨ A) 一个周末 (Recommended) ...
  B) 2-4 周的晚上 ...
```

差异：回答完目标和时间，AI 才会根据你的答案推荐合适的技术和规模——不是一开始就替你决定。

#### 场景 D：重构一个 module，但 AI 直接动手改了

**Before — AI 选了一种重构策略就开始改 code：**

```
你：这个 module 太肥了，帮我重构

AI：好的，我会把这个 module 拆成三个子模块：core、utils、types。
先从搬移 helper functions 开始...
```

问题：你只是觉得「太肥了」，AI 就决定了拆法。也许你想要的是抽 interface、或者只是搬几个函数出去，不需要大拆。

**After — AI 先问你想怎么拆：**

```
Q1: 重构的目标是什么？

  ✨ A) 降低单文件行数 (Recommended)
     把超过 500 行的文件拆成多个小文件
     😃：最小改动，风险低
     😫：只解决表面问题，依赖关系没变

  B) 厘清职责边界
     按功能拆成独立 module，定义清楚的 interface
     😃：架构变干净，未来好维护
     😫：改动范围大，要更新所有 import

  C) 抽出共用逻辑
     只把重复的部分抽成 shared utils
     😃：改动最少、立刻见效
     😫：没解决 module 本身的结构问题
```

差异：AI 没有直接开始拆 code，而是先确认你要的「重构」是哪一种。

### 价值二：AI 回了一大段分析，你读不完——帮你收拢成选择题

> AI 给了很多信息，但决策点散落在长文各处。这个 skill 把它们抽出来，变成一张清单。

---

#### 场景 E：AI 分析了三个行销渠道，你读完还是不知道选哪个

**Before — 决策点埋在散文里：**

```
AI：关于推广渠道，Facebook 广告的 CPC 大概在 $0.5-2 之间，适合
B2C 产品，但近年触及率下降明显。Google Ads 的意图比较明确，
转换率通常更高，不过竞争激烈的关键字 CPC 可能到 $5 以上。
如果你的受众是年轻族群，TikTok 的 CPM 目前还很低，但内容
制作门槛比较高，而且算法变动大...

你觉得哪个比较适合？
```

问题：你读了三遍，还是不确定三个渠道到底差在哪、哪个适合你。

**After — 同样的分析，收拢成一个选择题：**

```
Q1: 主要推广渠道选哪个？

  ✨ A) Google Ads (Recommended)
     搜索意图明确，用户主动找你
     😃：转换率高、效果可追踪
     😫：热门关键字贵（CPC $5+）

  B) Facebook 广告
     触及广、适合 B2C 品牌曝光
     😃：受众定位精准、素材格式多
     😫：触及率逐年下降、年轻人少用

  C) TikTok
     年轻族群集中、目前流量红利大
     😃：CPM 低、爆红机会高
     😫：内容制作门槛高、算法不稳定
```

差异：三秒看完三个渠道的取舍，直接选。

#### 场景 F：AI 写了一长段旅游规划，你不知道它在问你什么

**Before — 信息量大但没有结构：**

```
AI：京都五天的行程，第一天可以去清水寺和祇园，不过如果你对神社
比较有兴趣，伏见稻荷大社也在附近。交通的话建议买巴士一日券，
但如果你会去岚山，买 JR Pass 可能更划算。住宿方面四条河原町
最方便，不过东山区比较有气氛，就是离车站远一点...

你想怎么安排？
```

**After — 把你需要决定的事情拆出来：**

```
Q1: 行程风格偏好？
  ✨ A) 经典必去 (Recommended)
     清水寺、金阁寺、伏见稻荷，观光客路线
     😃：第一次去不会错过重点
     😫：人多、排队

  B) 在地体验
     町家咖啡、市场逛街、和服散步
     😃：悠闲、拍照好看
     😫：可能觉得「没去到什么景点」

Q2: 住宿区域？
  ✨ A) 四条河原町 (Recommended) ...
  B) 东山区 ...
```

#### 场景 G：AI 回了一大段技术分析，你读不完

**Before — 决策点埋在散文里，你要自己找：**

```
AI：关于数据库的选择，PostgreSQL 是最成熟的关联式数据库，支持 JSONB
可以处理半结构化数据，但如果你的数据量不大，SQLite 其实就够了，
它不需要额外的 server process，开发体验更好。不过如果未来要
多人同时写入的话，SQLite 的 write lock 会是瓶颈。另外你也可以
考虑用 Supabase，它底层是 PostgreSQL 但提供了即时 API 和
authentication，不过会多一层供应商依赖...

请问你倾向哪个方向？
```

问题：你读了三遍，还是不确定到底有几个选择、各自差在哪。

**After — 同样的分析，收拢成一个选择题：**

```
Q1: 数据库选哪个？

  ✨ A) SQLite (Recommended)
     嵌入式，不用另外装 DB server
     😃：零设置、开发最快、适合单人项目
     😫：多人同时写入会撞锁

  B) PostgreSQL
     最成熟的关联式数据库
     😃：功能完整、JSONB 支持好、未来好扩展
     😫：要装 server，开发环境多一步

  C) Supabase
     PostgreSQL + 即时 API + Auth 全包
     😃：快速有后端，不用自己写 API
     😫：多一层供应商依赖、免费额度有限
```

差异：同样的信息量，但你三秒就能看完所有选项的取舍，直接选。

#### 场景 H：CI pipeline 挂了，AI 丢了一堆可能原因

**Before — AI 列了五个可能，你不知道从哪个查起：**

```
AI：CI 失败的原因可能是：npm install 的 lock file 不一致、
Node 版本跟本地不同、有个 test 用了真实 API 但 CI 没有
credentials、也可能是 ESLint 的新 rule 挡住了、或者是上一次
merge 引入了 type error。建议你先看 CI log...
```

**After — 按优先顺序排好，问你从哪个开始查：**

```
Q1: 从哪个方向开始排查？

  ✨ A) 看 CI log 的第一个错误 (Recommended)
     直接看报错信息，不猜
     😃：最准确、不浪费时间
     😫：log 很长的话要花时间找

  B) 比对本地 vs CI 的 Node 版本
     跑 node -v 对照 CI config
     😃：30 秒就能排除
     😫：如果不是版本问题就白查了

  C) 检查最近一次 merge 的 diff
     看上一个 PR 改了什么
     😃：如果是 regression 一眼就找到
     😫：diff 大的话不容易看
```

## 怎么触发？

| 方式 | Claude Code | Codex CLI |
|---|---|---|
| **自动触发** | AI 遇到需要你决定的事情时，主动用结构化格式问你 | 同左 |
| **手动触发** | 输入 `/structured-questions` | 输入 `$structured-questions` |
| **关键字** | 对话中出现「帮我选」「不确定」「有几种做法」时也会触发 | 同左 |

## 适用场景

| 场景 | 你遇到的情况 | AI 怎么帮你 |
|---|---|---|
| **想法还很模糊** | 「我想办活动 / 做 side project / 经营自媒体，但不知道怎么开始」 | 帮你拆成目标、范围、形式三个问题，回答完方向就清楚 |
| **AI 回了长文** | 读完一大段技术分析 / 行销策略 / 旅游规划，不知道重点 | 把散文里的决策点抽出来，变成带优缺点的选择题 |
| **不知道有哪些选择** | AI 直接帮你选了，你连「原来还有别的选项」都不知道 | 列出所有可行方案，让你看到全貌再决定 |
| **排查问题** | CI 挂了 / bug 不知道从哪查起，AI 丢了一堆可能原因 | 按优先顺序排好，问你从哪个开始查 |
| **方向分歧** | 你和 AI 对「怎么做」有不同想法 | 把两边的方案并列，附优缺点，让你有依据做选择 |

## 不适用的情况

- 只有一种合理做法 → AI 直接做，不浪费你时间
- 你已经在指令中说清楚要怎么做 → AI 照做，不反问
- 纯进度回报 → 用文字说就好
