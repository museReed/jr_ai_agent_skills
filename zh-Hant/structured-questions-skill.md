# Structured Questions Skill — 讓 AI 幫你問出你自己問不出來的問題

**適用工具**: Claude Code / Codex CLI
**安裝指南**: [structured-questions-install.md](./structured-questions-install.md)

---

## 這個 Skill 解決什麼問題？

你有沒有遇過這種情況——

你叫 AI 幫你做一件事，它回了一大段文字，中間埋了三、四個需要你決定的事情，但你讀完整段之後，反而更不確定該怎麼回覆。或者更常見的：**你根本不知道這件事有什麼好問的，AI 就直接幫你決定了。**

這個 skill 做兩件事：

1. **讓 AI 主動幫你梳理思路。** 當你自己都還沒想清楚要什麼的時候，AI 會把混亂的局面拆成幾個具體的選擇題，每題附上選項、優缺點、和它的推薦——你不需要自己想出「該問什麼」，AI 幫你問。

2. **把散落在長文裡的決策點，收攏成一張清單。** 不再是「請看完以上分析後告訴我你的想法」這種丟球式回覆。每個需要你拍板的事情獨立列出，選項帶優缺點，推薦項標出來——你掃一眼就能做決定。

---

## Before / After 對比

### 價值一：你還沒想清楚的時候，AI 幫你拆解問題

> 你不需要先想好「該問什麼」——AI 主動把模糊的想法拆成具體的選擇題。

---

#### 場景 A：想辦一場活動，但腦中只有「想辦」兩個字

**Before — AI 直接幫你決定一切：**

```
你：幫我規劃一場活動

AI：好的！我建議辦一場 30 人的週末工作坊，地點選在共享空間，
主題是「AI 工具入門」。我先幫你列出時間表...
```

問題：你只是「想辦活動」，AI 就決定了人數、形式、主題、場地。你連「原來活動可以有這些類型」都沒想過。

**After — AI 幫你從混亂中理出頭緒：**

```
Q1: 這場活動的核心目標是什麼？

  ✨ A) 教學 / 知識分享 (Recommended)
     讓參加者學會一個具體技能
     😃：內容好準備，效果可衡量
     😫：需要你自己有足夠專業度

  B) 社群經營 / 認識人
     讓同領域的人互相連結
     😃：輕鬆、參加者壓力低
     😫：效果難量化，容易流於閒聊

  C) 品牌曝光 / 行銷
     讓更多人知道你或你的產品
     😃：直接帶來潛在客戶
     😫：需要行銷預算和推廣管道

Q2: 預計多少人參加？
  ✨ A) 10-15 人 (Recommended) ...
  B) 30-50 人 ...
  C) 100+ 人 ...
```

差異：你本來只有「想辦活動」。AI 幫你釐清「目標」「人數」「形式」三件事——回答完，你就知道該辦什麼了。

---

#### 場景 B：想開始經營自媒體，但不知道從哪裡開始

**Before — AI 替你選了平台和方向：**

```
你：我想開始經營自媒體

AI：建議你從 Instagram 開始，每週發 3 篇 Reels，主題聚焦在你的
專業領域。我幫你規劃第一週的內容排程...
```

問題：你還在想「要不要做」，AI 已經在排第一週的發文了。

**After — AI 先幫你想清楚幾個關鍵問題：**

```
Q1: 經營自媒體的主要動機是什麼？

  ✨ A) 建立個人品牌 (Recommended)
     讓業界知道你是誰、你擅長什麼
     😃：長期累積，對職涯有複利效果
     😫：需要持續產出，短期看不到回報

  B) 變現 / 賺錢
     透過業配、課程、付費內容獲利
     😃：有明確的收入目標可以追蹤
     😫：初期沒流量時收入幾乎為零

  C) 紀錄與分享
     把學到的東西寫下來，順便幫助別人
     😃：壓力最低，做得開心就好
     😫：成長速度慢，容易半途而廢

Q2: 你比較擅長哪種內容形式？
  ✨ A) 寫文字 (Recommended) ...
  B) 拍影片 ...
  C) 錄 Podcast ...
```

差異：AI 沒有直接叫你「去開 IG 帳號」，而是先幫你想清楚「為什麼做」和「用什麼形式」。

---

#### 場景 C：想做一個 Side Project，但還沒想清楚方向

**Before — AI 直接幫你決定做什麼、用什麼技術：**

```
你：幫我做一個 side project

AI：好的！我建議做一個個人部落格網站。我會用 Next.js + MDX 來建，
搭配 Tailwind CSS 做樣式，部署到 Vercel。先初始化專案...
```

問題：你只是想做「一個 side project」，AI 決定了產品、框架、部署。你連「原來還有這些選擇」都不知道。

**After — AI 幫你把模糊的念頭拆成三個選擇題：**

```
Q1: 這個 Side Project 的目標是什麼？

  ✨ A) 學新技術 (Recommended)
     做完學會一個新框架或語言
     😃：成就感高，履歷有東西寫
     😫：可能做不完

  B) 解決自己的痛點
     做一個自己每天會用的小工具
     😃：有動力做完，因為自己要用
     😫：範圍容易膨脹

  C) 建立作品集
     做一個能展示給面試官看的東西
     😃：求職直接加分
     😫：需要花時間打磨 UI

Q2: 你願意投入多少時間？
  ✨ A) 一個週末 (Recommended) ...
  B) 2-4 週的晚上 ...
```

差異：回答完目標和時間，AI 才會根據你的答案推薦合適的技術和規模——不是一開始就替你決定。

---

#### 場景 D：重構一個 module，但 AI 直接動手改了

**Before — AI 選了一種重構策略就開始改 code：**

```
你：這個 module 太肥了，幫我重構

AI：好的，我會把這個 module 拆成三個子模組：core、utils、types。
先從搬移 helper functions 開始...
```

問題：你只是覺得「太肥了」，AI 就決定了拆法。也許你想要的是抽 interface、或者只是搬幾個函數出去，不需要大拆。

**After — AI 先問你想怎麼拆：**

```
Q1: 重構的目標是什麼？

  ✨ A) 降低單檔行數 (Recommended)
     把超過 500 行的檔案拆成多個小檔
     😃：最小改動，風險低
     😫：只解決表面問題，依賴關係沒變

  B) 釐清職責邊界
     按功能拆成獨立 module，定義清楚的 interface
     😃：架構變乾淨，未來好維護
     😫：改動範圍大，要更新所有 import

  C) 抽出共用邏輯
     只把重複的部分抽成 shared utils
     😃：改動最少、立刻見效
     😫：沒解決 module 本身的結構問題
```

差異：AI 沒有直接開始拆 code，而是先確認你要的「重構」是哪一種。

### 價值二：AI 回了一大段分析，你讀不完——幫你收攏成選擇題

> AI 給了很多資訊，但決策點散落在長文各處。這個 skill 把它們抽出來，變成一張清單。

---

#### 場景 E：AI 分析了三個行銷管道，你讀完還是不知道選哪個

**Before — 決策點埋在散文裡：**

```
AI：關於推廣管道，Facebook 廣告的 CPC 大概在 $0.5-2 之間，適合
B2C 產品，但近年觸及率下降明顯。Google Ads 的意圖比較明確，
轉換率通常更高，不過競爭激烈的關鍵字 CPC 可能到 $5 以上。
如果你的受眾是年輕族群，TikTok 的 CPM 目前還很低，但內容
製作門檻比較高，而且演算法變動大...

你覺得哪個比較適合？
```

問題：你讀了三遍，還是不確定三個管道到底差在哪、哪個適合你。

**After — 同樣的分析，收攏成一個選擇題：**

```
Q1: 主要推廣管道選哪個？

  ✨ A) Google Ads (Recommended)
     搜尋意圖明確，用戶主動找你
     😃：轉換率高、效果可追蹤
     😫：熱門關鍵字貴（CPC $5+）

  B) Facebook 廣告
     觸及廣、適合 B2C 品牌曝光
     😃：受眾定位精準、素材格式多
     😫：觸及率逐年下降、年輕人少用

  C) TikTok
     年輕族群集中、目前流量紅利大
     😃：CPM 低、爆紅機會高
     😫：內容製作門檻高、演算法不穩定
```

差異：三秒看完三個管道的取捨，直接選。

#### 場景 F：AI 寫了一長段旅遊規劃，你不知道它在問你什麼

**Before — 資訊量大但沒有結構：**

```
AI：京都五天的行程，第一天可以去清水寺和祇園，不過如果你對神社
比較有興趣，伏見稻荷大社也在附近。交通的話建議買巴士一日券，
但如果你會去嵐山，買 JR Pass 可能更划算。住宿方面四條河原町
最方便，不過東山區比較有氣氛，就是離車站遠一點...

你想怎麼安排？
```

**After — 把你需要決定的事情拆出來：**

```
Q1: 行程風格偏好？
  ✨ A) 經典必去 (Recommended)
     清水寺、金閣寺、伏見稻荷，觀光客路線
     😃：第一次去不會錯過重點
     😫：人多、排隊

  B) 在地體驗
     町家咖啡、市場逛街、和服散步
     😃：悠閒、拍照好看
     😫：可能覺得「沒去到什麼景點」

Q2: 住宿區域？
  ✨ A) 四條河原町 (Recommended) ...
  B) 東山區 ...
```

#### 場景 G：AI 回了一大段技術分析，你讀不完

**Before — 決策點埋在散文裡，你要自己找：**

```
AI：關於資料庫的選擇，PostgreSQL 是最成熟的關聯式資料庫，支援 JSONB
可以處理半結構化資料，但如果你的資料量不大，SQLite 其實就夠了，
它不需要額外的 server process，開發體驗更好。不過如果未來要
多人同時寫入的話，SQLite 的 write lock 會是瓶頸。另外你也可以
考慮用 Supabase，它底層是 PostgreSQL 但提供了即時 API 和
authentication，不過會多一層供應商依賴...

請問你傾向哪個方向？
```

問題：你讀了三遍，還是不確定到底有幾個選擇、各自差在哪。

**After — 同樣的分析，收攏成一個選擇題：**

```
Q1: 資料庫選哪個？

  ✨ A) SQLite (Recommended)
     嵌入式，不用另外裝 DB server
     😃：零設定、開發最快、適合單人專案
     😫：多人同時寫入會撞鎖

  B) PostgreSQL
     最成熟的關聯式資料庫
     😃：功能完整、JSONB 支援好、未來好擴展
     😫：要裝 server，開發環境多一步

  C) Supabase
     PostgreSQL + 即時 API + Auth 全包
     😃：快速有後端，不用自己寫 API
     😫：多一層供應商依賴、免費額度有限
```

差異：同樣的資訊量，但你三秒就能看完所有選項的取捨，直接選。

#### 場景 H：CI pipeline 掛了，AI 丟了一堆可能原因

**Before — AI 列了五個可能，你不知道從哪個查起：**

```
AI：CI 失敗的原因可能是：npm install 的 lock file 不一致、
Node 版本跟本地不同、有個 test 用了真實 API 但 CI 沒有
credentials、也可能是 ESLint 的新 rule 擋住了、或者是上一次
merge 引入了 type error。建議你先看 CI log...
```

**After — 按優先順序排好，問你從哪個開始查：**

```
Q1: 從哪個方向開始排查？

  ✨ A) 看 CI log 的第一個錯誤 (Recommended)
     直接看報錯訊息，不猜
     😃：最準確、不浪費時間
     😫：log 很長的話要花時間找

  B) 比對本地 vs CI 的 Node 版本
     跑 node -v 對照 CI config
     😃：30 秒就能排除
     😫：如果不是版本問題就白查了

  C) 檢查最近一次 merge 的 diff
     看上一個 PR 改了什麼
     😃：如果是 regression 一眼就找到
     😫：diff 大的話不容易看
```

## 怎麼觸發？

| 方式 | Claude Code | Codex CLI |
|---|---|---|
| **自動觸發** | AI 遇到需要你決定的事情時，主動用結構化格式問你 | 同左 |
| **手動觸發** | 輸入 `/structured-questions` | 輸入 `$structured-questions` |
| **關鍵字** | 對話中出現「幫我選」「不確定」「有幾種做法」時也會觸發 | 同左 |

## 適用場景

| 場景 | 你遇到的情況 | AI 怎麼幫你 |
|---|---|---|
| **想法還很模糊** | 「我想辦活動 / 做 side project / 經營自媒體，但不知道怎麼開始」 | 幫你拆成目標、範圍、形式三個問題，回答完方向就清楚 |
| **AI 回了長文** | 讀完一大段技術分析 / 行銷策略 / 旅遊規劃，不知道重點 | 把散文裡的決策點抽出來，變成帶優缺點的選擇題 |
| **不知道有哪些選擇** | AI 直接幫你選了，你連「原來還有別的選項」都不知道 | 列出所有可行方案，讓你看到全貌再決定 |
| **排查問題** | CI 掛了 / bug 不知道從哪查起，AI 丟了一堆可能原因 | 按優先順序排好，問你從哪個開始查 |
| **方向分歧** | 你和 AI 對「怎麼做」有不同想法 | 把兩邊的方案並列，附優缺點，讓你有依據做選擇 |

## 不適用的情況

- 只有一種合理做法 → AI 直接做，不浪費你時間
- 你已經在指令中說清楚要怎麼做 → AI 照做，不反問
- 純進度回報 → 用文字說就好
